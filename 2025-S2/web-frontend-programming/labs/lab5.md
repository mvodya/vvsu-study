# Лабораторная работа №5 - Адаптивная верстка

## Задание

Создать страницу `profile.html` - адаптивный профиль пользователя (социальная сеть, блог, игровое сообщество и т.п.), оформленный с помощью Tailwind CSS. Для примера можно взять реальную социальную сеть, или выдумать свою. Допускается в качестве пользователя использовать не человека (котика, зомби, покемона)

Страница должна автоматически перестраиваться под разные экраны: смартфон, планшет и компьютер

## Требования

### 1. Установка и настройка проекта (20%)

* Проект должен быть создан и настроен аналогично лабораторной №4:

  * установка Tailwind CSS через Node.js (`npm init -y`, `npm i -D tailwindcss @tailwindcss/cli`)
  * корректная структура (`src/`, `dist/`, `index.html` / `profile.html`)
  * сборка CSS с помощью `npx @tailwindcss/cli -i ./src/app.css -o ./dist/output.css --watch`
  * подключение итогового файла `output.css` в HTML

### 2. Адаптивная страница профиля (60%)

Создать страницу с адаптивной версткой, содержащую следующие элементы:

Обязательные блоки:

1. Шапка (header)

     * Логотип или название сервиса
     * Меню (например: "Главная", "Сообщения", "Настройки")
     * На мобильных устройствах меню должно превращаться в компактное (или скрываться под кнопкой)

2. Основной блок профиля

     * Аватар пользователя (круглая или квадратная фотография)
     * Имя и краткая информация (город, профессия, статус)
     * Кнопки действий - "Подписаться", "Написать", "Редактировать"
     * Адаптивное поведение:

       * На телефоне - элементы в колонку
       * На планшете и выше - аватар и данные в строку

3. Контент профиля

     * Раздел "Публикации", "Фотографии" или "Обо мне"
     * Карточки или сетка публикаций
     * Сетка должна быть адаптивной:

       * `1` колонка на телефоне
       * `2` на планшете
       * `3–4` на десктопе

4. Боковая панель (по желанию)

     * Список друзей, подписчиков, теги или статистика
     * Можно скрывать ее на мобильных устройствах (`hidden md:block`)

5. Футер (footer)

     * Копирайт, контакты или ссылки
     * Центрирование текста и адаптивное выравнивание

Адаптивные требования:

* Использовать **брейкпоинты Tailwind** (`sm:`, `md:`, `lg:`, `xl:`)
* Применить **flex** или **grid** для организации макета
* Изображения должны масштабироваться (`w-full h-auto`)
* Текст и кнопки - увеличиваться на больших экранах
* Проверить корректное отображение при ширине 375 px, 768 px и 1280 px

### 3. Соблюдение визуального стиля (20 %)

* Страница должна соответствовать **одному выбранному стайл-гайду**:

  * **Material Design** (Google)
  * **Apple HIG**
  * **Fluent Design** (Microsoft)

* Цветовая палитра, типографика и отступы должны быть согласованы между всеми элементами
