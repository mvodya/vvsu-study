# Адаптивная верстка

## Что такое адаптивная верстка

**Адаптивная верстка** - это способ создания веб-страниц, при котором дизайн автоматически подстраивается под размер и особенности экрана: смартфона, планшета, ноутбука или большого монитора. Цель адаптивности - сделать сайт удобным и красивым на любом устройстве, без горизонтальной прокрутки, налезания элементов и съехавших блоков

Проще говоря - адаптивная верстка делает сайт гибким: он **меняет расположение и размеры элементов**, чтобы сохранить читаемость и удобство

### Пример

* На **компьютере** блоки могут располагаться в три колонки
* На **планшете** - уже в две
* А на **смартфоне** - превращаются в один вертикальный список

При этом контент остается тем же - меняется только его расположение и размер

### Зачем нужна адаптивность

1. **Половина трафика - с мобильных устройств**

   По статистике, более 60% пользователей просматривают сайты со смартфонов. Если сайт неудобен на маленьком экране, пользователь просто уходит

2. **SEO и поисковая выдача**

   Поисковики (Google, Яндекс) понижают сайты без адаптивности. Удобная мобильная версия - обязательный критерий mobile-friendly

3. **Удобство и доступность**

   Адаптивные страницы обеспечивают хорошую читаемость текста, удобные кнопки, логичную навигацию и отсутствие горизонтального скролла - все это делает сайт доступным для всех пользователей

4. **Единая кодовая база**

   Адаптивная верстка позволяет **не создавать отдельные сайты** для разных устройств (например, `m.site.ru`), а просто адаптировать один дизайн под разные размеры

### Отличие от других подходов

| Подход                      | Особенности                                                            | Пример                                                  |
| --------------------------- | ---------------------------------------------------------------------- | ------------------------------------------------------- |
| **Фиксированная верстка**   | Размеры заданы в пикселях, страница не меняется при изменении экрана   | Сайт обрезается или появляется горизонтальная прокрутка |
| **Резиновая (fluid)**       | Элементы растягиваются пропорционально ширине окна                     | При очень широком экране может расползаться             |
| **Адаптивная (responsive)** | Макет перестраивается при достижении определенных ширин (брейкпоинтов) | Оптимальный вариант для современных сайтов              |

### Принцип mobile first

Современные сайты создаются по принципу **mobile-first** - "сначала для мобильных устройств"

Это значит, что **основная версия сайта разрабатывается для смартфонов**, а затем к ней добавляются стили для планшетов и больших экранов. Такой подход делает код проще, логичнее и снижает вероятность ошибок: сайт гарантированно хорошо выглядит на мобильных устройствах, а не наоборот

## Основные принципы адаптивного дизайна

Чтобы сайт был адаптивным, недостаточно просто уменьшить шрифты или сузить блоки. Нужно проектировать интерфейс так, чтобы он оставался удобным и логичным при любой ширине экрана. Для этого используются несколько ключевых принципов

### 1. Использование гибких сеток (Grid и Flexbox)

Вместо фиксированных позиций и размеров элементы размещают в **гибких контейнерах**

Это могут быть:

* **Flexbox** - гибкое распределение блоков по горизонтали или вертикали
* **CSS Grid** - система для построения таблиц и карточек с адаптивным поведением

Пример (Tailwind CSS):

```html
<div class="grid grid-cols-1 md:grid-cols-2 lg:grid-cols-3 gap-6">
  <div class="bg-blue-200 p-4 rounded">Блок 1</div>
  <div class="bg-blue-200 p-4 rounded">Блок 2</div>
  <div class="bg-blue-200 p-4 rounded">Блок 3</div>
</div>
```

* На мобильных (`grid-cols-1`) - один столбец
* На планшетах (`md:grid-cols-2`) - два
* На больших экранах (`lg:grid-cols-3`) - три

### 2. Пропорциональные единицы измерения

Фиксированные размеры в пикселях (`px`) плохо подходят для адаптивной верстки

Лучше использовать **относительные единицы** - они автоматически масштабируются:

| Единица      | Описание                                 | Пример              |
| ------------ | ---------------------------------------- | ------------------- |
| `%`          | процент от родительского блока           | `width: 50%`        |
| `vw` / `vh`  | процент от ширины / высоты окна браузера | `width: 80vw`       |
| `em` / `rem` | относительно шрифта родителя / корня     | `font-size: 1.2rem` |

В Tailwind это выглядит так:

```html
<div class="w-1/2 h-[50vh] text-[1.2rem]">
  Пропорциональные размеры
</div>
```

### 3. Гибкие изображения и мультимедиа

Картинки, видео и другие элементы должны **автоматически подстраиваться под контейнер**, не выходя за его границы. Для этого достаточно задать им ширину `100%` и высоту `auto`

В Tailwind это реализуется просто:

```html
<img src="photo.jpg" class="w-full h-auto rounded-lg" alt="Пример">
```

* Картинка растянется на всю ширину родителя, сохранив пропорции
* Не выйдет за границы блока при уменьшении экрана

### 4. Минимум фиксированных значений

Фиксированные высоты и ширины (`width: 500px`, `height: 400px`) делают макет ломким. Если экран меньше - элементы выходят за пределы окна. Адаптивный дизайн требует гибкости: отступы, шрифты и размеры лучше задавать через утилитарные классы Tailwind (`p-4`, `text-lg`, `w-1/3`, и т.п.)

### 5. Иерархия и приоритет контента

На маленьком экране все не поместится, поэтому важно расставить приоритеты:

* Сначала основной контент (имя пользователя, аватар, кнопки действий)
* Потом - второстепенные элементы (дополнительные блоки, комментарии, рекомендации)
* В Tailwind можно скрывать или показывать части интерфейса по брейкпоинтам:

```html
<p class="hidden md:block text-gray-600">
  Это описание видно только на экране шире 768px
</p>
```

### 6. Простота и читаемость

Адаптивный сайт должен быть **простым и интуитивным**.

Шрифты - не слишком мелкие, кнопки - достаточно большие для нажатия пальцем, поля - с достаточными отступами. Слишком сложные композиции и анимации ухудшают восприятие на мобильных устройствах.

## Брейкпоинты и медиа-запросы

**Брейкпоинты (breakpoints)** - это точки перелома, при которых макет страницы меняет свое поведение, в зависимости от ширины экрана. Благодаря им сайт может выглядеть по-разному на телефоне, планшете и мониторе, оставаясь при этом единым проектом

### Что такое брейкпоинт

Брейкпоинт определяет **ширину экрана**, начиная с которой к элементам применяются дополнительные стили. Обычно их задают в пикселях (px), но браузер все делает автоматически: когда окно становится уже - макет перестраивается

Пример на чистом CSS:

```css
/* Основной стиль (для мобильных) */
.card {
  width: 100%;
}

/* При ширине >= 768px (планшет) */
@media (min-width: 768px) {
  .card {
    width: 48%;
  }
}

/* При ширине >= 1024px (ноутбук / десктоп) */
@media (min-width: 1024px) {
  .card {
    width: 32%;
  }
}
```

Таким образом, один и тот же элемент может иметь **три разных внешних вида**

### Классические диапазоны экранов

Веб-разработчики часто используют стандартные диапазоны для разных устройств:

| Устройство         | Диапазон ширины | Пример брейкпоинта |
| ------------------ | --------------- | ------------------ |
| Смартфоны          | до **640px**    | `sm`               |
| Планшеты           | от **768px**    | `md`               |
| Небольшие ноутбуки | от **1024px**   | `lg`               |
| Мониторы           | от **1280px**   | `xl`               |
| Большие дисплеи    | от **1536px**   | `2xl`              |

Эти значения - стандарт **Tailwind CSS**, поэтому нам не нужно прописывать `@media` вручную: фреймворк уже знает, какой класс применять на каком экране

### Мобильный-first подход

Tailwind использует mobile-first модель: основные классы применяются по умолчанию для мобильных устройств, а префиксы (`sm:`, `md:`, `lg:`…) добавляют стили начиная с указанного размера и выше.

Это противоположно desktop-first, где стили уменьшаются при сужении экрана.

### Брейкпоинты в Tailwind CSS

Tailwind реализует адаптивность через префиксы к классам

Формат:

```
<брейкпоинт>:<класс>
```

#### Пример:

```html
<div class="text-sm sm:text-base md:text-lg lg:text-xl xl:text-2xl">
  Пример адаптивного текста
</div>
```

Что произойдет:

| Экран     | Размер текста |
| --------- | ------------- |
| до 640px  | `text-sm`     |
| >= 640px  | `text-base`   |
| >= 768px  | `text-lg`     |
| >= 1024px | `text-xl`     |
| >= 1280px | `text-2xl`    |

### Адаптивные примеры с Tailwind

#### Пример: Адаптивная сетка карточек

```html
<div class="grid grid-cols-1 sm:grid-cols-2 lg:grid-cols-3 gap-4">
  <div class="bg-slate-200 p-4 rounded">Карточка 1</div>
  <div class="bg-slate-200 p-4 rounded">Карточка 2</div>
  <div class="bg-slate-200 p-4 rounded">Карточка 3</div>
</div>
```

* На телефоне - 1 колонка
* На планшете - 2
* На ноутбуке - 3

#### Пример: Изменение выравнивания и отступов

```html
<div class="text-center md:text-left p-4 md:p-8">
  <p>Этот текст по центру на телефоне и выравнен слева на планшете</p>
</div>
```

#### Пример: Скрытие элементов

```html
<div class="block md:hidden">Мобильное меню</div>
<div class="hidden md:block">Меню для ПК</div>
```

На телефоне показывается только мобильное меню, а на большом экране - полноценная навигация

### Комбинирование классов

В Tailwind можно использовать несколько брейкпоинтов одновременно, создавая детально проработанную адаптивность:

```html
<div class="w-full sm:w-3/4 md:w-1/2 lg:w-2/5 xl:w-1/3">
  Адаптивная ширина блока
</div>
```

Таким образом, на каждом экране блок будет занимать разную долю ширины контейнера - от 100% до 33%

## Адаптивность в Tailwind CSS

Tailwind CSS - один из самых удобных инструментов для создания адаптивных интерфейсов. Он полностью основан на mobile-first подходе и предоставляет интуитивные утилиты для управления внешним видом элементов на разных размерах экрана - без написания `@media` вручную

Tailwind применяет стили снизу вверх: по умолчанию классы действуют на мобильных устройствах, а при достижении брейкпоинта добавляются новые

### Адаптивные макеты и сетки

Tailwind поддерживает как Flexbox, так и CSS Grid, и оба варианта можно комбинировать с брейкпоинтами

#### Пример: Flex-контейнер, который перестраивается

```html
<div class="flex flex-col md:flex-row gap-4">
  <div class="flex-1 bg-indigo-200 p-4 rounded">Левая панель</div>
  <div class="flex-1 bg-indigo-300 p-4 rounded">Правая панель</div>
</div>
```

* На телефоне - колонки располагаются вертикально (`flex-col`)
* На планшете и выше - становятся горизонтальными (`md:flex-row`)

#### Пример: адаптивная сетка

```html
<div class="grid grid-cols-1 sm:grid-cols-2 lg:grid-cols-4 gap-4">
  <div class="bg-gray-200 p-4 rounded">1</div>
  <div class="bg-gray-200 p-4 rounded">2</div>
  <div class="bg-gray-200 p-4 rounded">3</div>
  <div class="bg-gray-200 p-4 rounded">4</div>
</div>
```

Гибкая структура, автоматически меняющая количество колонок

### Работа с изображениями и медиаконтентом

Для изображений важно не только адаптивное масштабирование (`w-full h-auto`), но и умение изменять форму или выравнивание при разных размерах

#### Пример

```html
<img src="avatar.jpg"
     class="w-32 h-32 md:w-48 md:h-48 lg:w-64 lg:h-64 rounded-full mx-auto md:mx-0"
     alt="Аватар пользователя">
```

* На телефоне аватар по центру, размер небольшой
* На планшете - больше и выровнен по левому краю

### Перестройка структуры (порядок элементов)

Tailwind позволяет менять порядок элементов в потоке для разных экранов

```html
<div class="flex flex-col md:flex-row">
  <div class="order-2 md:order-1 bg-slate-300 p-4">Контент</div>
  <div class="order-1 md:order-2 bg-slate-200 p-4">Боковая панель</div>
</div>
```

На телефоне сначала боковая панель, затем контент. На больших экранах - наоборот. Это удобно, когда нужно менять визуальный приоритет блоков

### Скрытие и отображение элементов

Tailwind упрощает работу с адаптивным отображением:

```html
<div class="block md:hidden">Мобильное меню</div>
<div class="hidden md:block">Меню для ПК</div>
```

Можно также использовать `lg:flex` и `lg:hidden` для более тонкого контроля

### Пример адаптивного профиля (фрагмент)

```html
<div class="flex flex-col md:flex-row items-center md:items-start gap-6 p-6 bg-white rounded-lg shadow">
  <img src="avatar.jpg" class="w-32 h-32 md:w-40 md:h-40 rounded-full" alt="User photo">
  <div>
    <h1 class="text-2xl font-bold text-gray-800 md:text-3xl">Имя пользователя</h1>
    <p class="text-gray-600 mt-2 md:max-w-md">
      Короткое описание профиля. Этот текст может занимать несколько строк, но на мобильных устройствах он автоматически переносится.
    </p>
    <div class="mt-4 flex gap-3">
      <button class="bg-indigo-600 text-white px-4 py-2 rounded hover:bg-indigo-700 transition">Подписаться</button>
      <button class="bg-gray-200 text-gray-800 px-4 py-2 rounded hover:bg-gray-300 transition">Сообщение</button>
    </div>
  </div>
</div>
```

* На телефоне - все в колонку
* На планшете и ПК - фото и данные в ряд

## Сетки и макеты (layout)

Основу любого адаптивного сайта составляет макет (layout) - то есть расположение блоков на странице: шапка, меню, контент, боковые панели, футер и т.д. Tailwind CSS предоставляет инструменты для построения гибких и отзывчивых макетов с минимальным количеством кода

### Основные подходы к макету

В адаптивной верстке чаще всего используются два подхода:

| Подход       | Описание                                                                                         | Применение                                     |
| ------------ | ------------------------------------------------------------------------------------------------ | ---------------------------------------------- |
| **Flexbox**  | Элементы выстраиваются в ряд или колонку, можно управлять выравниванием, направлением и порядком | Простые строки, навигационные панели, карточки |
| **CSS Grid** | Макет делится на ячейки, можно задавать колонки и строки разных размеров                         | Комплексные сетки, галереи, страницы профиля   |

### Макеты на Flexbox

#### Пример: Простая структура страницы

```html
<div class="bg-indigo-600 text-white p-4 text-center">
  Заголовок сайта
</div>

<div class="flex flex-col md:flex-row">
  <aside class="bg-indigo-100 p-4 md:w-1/4">Меню</aside>
  <section class="bg-white p-6 md:w-3/4">Контент</section>
</div>

<div class="bg-gray-100 text-center p-4 text-gray-600">
  © 2025 MySite
</div>
```

* На телефоне - блоки идут вертикально
* На планшете и выше - меню и контент располагаются в две колонки

Tailwind делает макет гибким: достаточно использовать классы `flex`, `flex-col`, `md:flex-row` и относительные ширины `w-1/4`, `w-3/4`

#### Пример: Центрирование и адаптивные карточки

```html
<div class="flex flex-wrap justify-center gap-6 p-6">
  <div class="w-full sm:w-1/2 lg:w-1/3 bg-slate-200 p-4 rounded-lg shadow">Карточка 1</div>
  <div class="w-full sm:w-1/2 lg:w-1/3 bg-slate-200 p-4 rounded-lg shadow">Карточка 2</div>
  <div class="w-full sm:w-1/2 lg:w-1/3 bg-slate-200 p-4 rounded-lg shadow">Карточка 3</div>
</div>
```

* `flex-wrap` - позволяет элементам переноситься на новую строку
* `justify-center` - выравнивает карточки по центру
* `gap-6` - задает расстояние между ними

### Макеты на Grid

Grid идеально подходит для **профилей пользователей**, **галерей** и **страниц с множеством блоков**

Особенность Grid (в отличие от flexbox) - сохранение размеров ячеек одинаковыми (если для ячейки не прописаны дополнительные параметры)

#### Пример: Простая адаптивная сетка

```html
<div class="grid grid-cols-1 md:grid-cols-2 lg:grid-cols-4 gap-4 p-4">
  <div class="bg-gray-200 p-4 rounded">Блок 1</div>
  <div class="bg-gray-200 p-4 rounded">Блок 2</div>
  <div class="bg-gray-200 p-4 rounded">Блок 3</div>
  <div class="bg-gray-200 p-4 rounded">Блок 4</div>
</div>
```

* 1 колонка
* 2 колонки на планшете
* 4 колонки на большом экране

#### Пример: Макет профиля пользователя

```html
<div class="grid grid-cols-1 lg:grid-cols-3 gap-6 p-6">
  <!-- Левая панель (аватар, информация) -->
  <aside class="bg-white rounded-lg p-6 shadow">
    <img src="avatar.jpg" class="w-32 h-32 mx-auto rounded-full mb-4" alt="">
    <h2 class="text-xl font-bold text-center">Имя пользователя</h2>
    <p class="text-gray-600 text-center mt-2">Короткое описание профиля</p>
  </aside>

  <!-- Центральный контент (посты, фото, активность) -->
  <section class="lg:col-span-2 bg-white rounded-lg p-6 shadow">
    <h3 class="text-lg font-semibold mb-4">Последние публикации</h3>
    <div class="grid grid-cols-1 md:grid-cols-2 gap-4">
      <div class="bg-slate-100 p-4 rounded">Пост 1</div>
      <div class="bg-slate-100 p-4 rounded">Пост 2</div>
      <div class="bg-slate-100 p-4 rounded">Пост 3</div>
      <div class="bg-slate-100 p-4 rounded">Пост 4</div>
    </div>
  </section>
</div>
```

* На мобильном все идет вертикально
* На больших экранах - три колонки с независимыми зонами

### Центрирование и выравнивание

В Tailwind почти все задачи с выравниванием решаются через комбинацию классов:

| Цель                         | Классы                                       |
| ---------------------------- | -------------------------------------------- |
| Центрирование по горизонтали | `mx-auto`                                    |
| Центрирование по вертикали   | `flex items-center justify-center`           |
| Равномерное распределение    | `justify-between`, `justify-around`          |
| Выравнивание по сетке        | `place-items-center`, `place-content-center` |

#### Пример

```html
<div class="flex items-center justify-center h-screen bg-slate-100">
  <div class="bg-white p-8 rounded shadow">
    <h1 class="text-2xl font-bold text-gray-800">Центрированный блок</h1>
  </div>
</div>
```

### Работа с высотой и шириной

Tailwind имеет удобные классы для управления размерами:

| Свойство        | Пример                          | Описание                |
| --------------- | ------------------------------- | ----------------------- |
| `w-full`        | 100% ширины контейнера          | Растягивание            |
| `max-w-md`      | ограничение максимальной ширины | Полезно для текста      |
| `h-screen`      | 100% высоты окна                | Полноэкранные секции    |
| `min-h-[value]` | минимальная высота              | Для адаптивных карточек |

#### Пример

```html
<div class="max-w-lg mx-auto bg-white p-6 rounded shadow h-auto min-h-[200px]">
  Контент
</div>
```

### Отступы и поля (spacing)

Tailwind имеет удобную систему отступов `m-` и `p-` (margin / padding)

Они могут быть адаптивными:

```html
<div class="p-2 sm:p-4 md:p-8 lg:p-12">
  Контент с адаптивными отступами
</div>
```

* На телефоне - плотный макет
* На десктопе - больше пространства

## Практика адаптивности

Создать адаптивный сайт - это не только написать классы, но и проверить, как все реально работает на разных устройствах и разрешениях. Ниже - основные практические приемы и инструменты, которые помогут отладить и улучшить адаптивность вашего макета.

### Проверка в инструментах разработчика

Каждый современный браузер имеет инструменты для проверки адаптивности

#### Как открыть

* **Google Chrome / Microsoft Edge** - нажмите `F12` или `Ctrl + Shift + I`
* **Firefox** - `Ctrl + Shift + M`
* **Safari (macOS)** - меню *Разработка -> Показать инспектор веба*

#### Как пользоваться

В панели инструментов есть кнопка Toggle device toolbar (иконка телефона). Она позволяет переключаться между типами устройств:

* iPhone, Pixel, Samsung
* iPad, планшеты
* стандартный десктоп

При этом можно:

* выбирать предустановленные устройства (например, iPhone 14 Pro)
* вручную задавать ширину и высоту окна
* поворачивать экран (портрет / ландшафт)

> Проверяйте не только адаптивность, но и удобство - не слишком ли мелкие кнопки, помещается ли текст, видны ли основные элементы без прокрутки

### Тестирование на реальных устройствах

Никакая эмуляция не заменит реального теста

Если есть возможность, **проверьте страницу на телефоне**: откройте свой HTML-файл через локальный сервер (`Live Server` в VS Code, или через `python3 -m http.server 8080`)

* Проверяйте скорость загрузки и отклик кнопок
* Тестируйте горизонтальную ориентацию
* Обращайте внимание на тач-интерфейс - кнопки не должны быть слишком близко

### Изображения и адаптивные размеры

В адаптивной верстке изображения должны:

1. Масштабироваться под ширину контейнера (`class="w-full h-auto"`)
2. Иметь разумное соотношение сторон (например, квадрат для аватара)
3. Не "прыгать" при изменении ориентации экрана

В Tailwind достаточно задать:

```html
<img src="profile.jpg" class="w-full sm:w-1/2 lg:w-1/3 h-auto rounded-lg mx-auto" alt="Фото профиля">
```

Такое изображение:

* занимает всю ширину на телефоне
* половину - на планшете
* треть - на десктопе

### 4Шрифты и читаемость

Главное (и ооочень простое) правило адаптивной типографики - текст должен оставаться читаемым. В Tailwind можно задавать разные размеры текста для разных экранов:

```html
<p class="text-base sm:text-lg md:text-xl lg:text-2xl leading-relaxed">
  Пример адаптивного текста, который увеличивается на больших экранах
</p>
```

Также полезно:

* Увеличивать межстрочный интервал (`leading-relaxed`)
* Избегать длинных строк (оптимально ~60–70 символов в строке)
* Задавать максимальную ширину блока (`max-w-prose` или `max-w-lg`)

### Тестирование поведения сетки

Иногда при изменении ширины окна сеточные элементы ломаются - выравнивание теряется, блоки налезают

Чтобы это избежать:

* Используйте `gap-*` вместо `margin` между элементами
* Не задавайте фиксированные высоты (`h-64`, `h-80` и т.д.), если не уверены
* Проверяйте перестройку при каждой "точке перелома" (`sm`, `md`, `lg`, `xl`)

> Хороший прием - временно раскрасить блоки (`bg-red-200`, `bg-green-200`) чтобы наглядно увидеть, как они перестраиваются при изменении ширины.

### Удобство взаимодействия

На мобильных устройствах:

* кнопки должны быть не меньше **40×40 пикселей** (`py-2 px-4` или `py-3 px-6`)
* поля ввода - с увеличенными отступами
* ссылки - не расположены слишком близко

Пример адаптивной кнопки:

```html
<button class="bg-indigo-600 text-white text-sm sm:text-base px-4 py-2 sm:px-6 sm:py-3 rounded-lg hover:bg-indigo-700 transition">
  Отправить сообщение
</button>
```

### Отладка и оптимизация

Если адаптивность не срабатывает, проверьте:

1. Есть ли префикс брейкпоинта (`md:`, `lg:` и т.д.)
2. Не перекрывается ли стиль другим классом (в Tailwind порядок классов не имеет значения, но брейкпоинты приоритетнее)
3. Используется ли актуальная версия Tailwind (версии до 3.0 имели иные правила)
4. Не ограничена ли ширина родительского контейнера (`max-w-*`, `overflow-hidden`)
